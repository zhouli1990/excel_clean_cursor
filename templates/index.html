表格数据清洗

上传一个或多个Excel（.xlsx、.xls）或CSV（.CSV）文件，并拉取飞书多维表格数据以进行整合和标准化，最后更新飞书多维表格。

基本流程:

  1. (可选) 点击下方的 "配置选项" 展开，根据需要调整参数，完成后可点击 "保存所有设置为默认值"。
  2. (必要) 第一步：上传需要处理的 Excel 或 CSV 文件。
  3. (必要) 第二步：点击 "开始处理" 按钮。等待处理进度更新。处理完成后下载最终结果文件。
  4. (必要) 第三步：进入 "人工检查与调整"，下载文件并在本地修改。
  5. (必要) 第四步：上传调整后数据，选择并上传您修改后的文件。
  6. (可选) 第五步：检查差异，点击 "检查差异" 按钮，会展示您修改的内容，用作辅助验证。
  7. (必要) 第六步：确认差异无误后，点击 "同步到飞书" 将更改应用到飞书表格。

注意事项:

  • 配置：请确保您的 API Key 正确且有足够的额度。
  • 配置：目标列名请每行输入一个。程序会尝试从源文件中提取信息并填充到这些列中。
  • 配置：目标列名中`来源`请勿删除。
  • 配置：后处理操作的列名选择依赖于 "目标列名" 的设置。请务必确保目标列名中的字段在飞书的表单中存在此字段，否则会无法同步。
  • 配置：批处理大小 (Batch Size) 影响每次调用 API 处理的行数。较大的值可能更快，但也更消耗 Token 和内存，并可能超过 API 的单次请求限制，默认160，不建议再大于此值。
  • 配置：如果遇到 "Context length exceeded" 错误，请尝试减小批处理大小。
  • 配置：飞书数据源配置-->新增目标 Table ID(s):此处配置决定了新增数据同步至哪张多维表格。系统在新增数据时，会首先尝试写入第一个 Table ID。如果检查发现写入后会超过该表格的行数上限 (企业版最大为 50000 行)，则会自动尝试列表中的下一个 Table ID，以此类推。如果所有指定的目标表格都已满，新增操作将失败并报错。如果未配置此项，新增数据将写入 "飞书 Table IDs" 列表中的第一个表格。
  • 第三步：`local_row_id` 列与 `record_id` 请勿手动修改，是用作更新的主键。
  • 第四步：需上传系统生成的final_*.xlsx的文件，请勿上传其他文件
